### Greeting/subtitle width diagnosis

The greeting and subtitle were still appearing constrained. The issue was first suspected to be the `AdminWelcomeName` component or the HTML structure. Inspection showed `AdminWelcomeName.tsx` returns plain text only:

```tsx
return <>{name}</>;
```

So the component was not forcing line breaks.

The actual issue was multiple CSS constraints:

1. `text-wrap: balance` on the greeting was causing artificially balanced shorter lines.
2. A later-loaded `admin-polish-overrides.scss` file still capped dashboard text widths.
3. That later file used global mobile rules such as:

```css
.bp-dashboard h1 {
  max-width: 18rem !important;
}

.bp-dashboard p {
  max-width: 21rem !important;
}
```

This affected more than the greeting:

- Welcome heading.
- Live order subtitle.
- System Status subtitle.
- Recent Orders subtitle.
- Launch Funnel subtitle.
- Community Growth subtitle.
- Other dashboard card paragraph text.

Related commits:

### Card row/data layout diagnosis

After the text-width caps were removed, the user noticed that the dashboard data rows themselves were still stacked. This affected the highlighted empty right-side space in:

- System Status rows.
- Recent Orders rows.
- Community Growth subscriber rows.

The cause was the mobile CSS in `BeforeDashboard.scss` forcing rows to one-column layouts:

```css
.bp-dashboard__orderRow { grid-template-columns: minmax(0, 1fr); }
.bp-dashboard__subscriberRow { grid-template-columns: minmax(0, 1fr); }
```

The System Status row was also being forced to place the `ONLINE` badge underneath instead of in the third column.

A new final stylesheet was created and imported last in the Payload admin layout so it wins over earlier mobile stacking rules:

```txt
app/(payload)/admin-dashboard-mobile-rows.scss
app/(payload)/layout.tsx
```

Commits:

```txt

a55194a092729fcd645070019cb35c01a12498e3
Load mobile dashboard row overrides last
```

Goal of this fix:

```txt
Outer dashboard cards stay single-column/mobile-friendly.
```
