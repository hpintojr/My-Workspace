- Total.
- Purchased item list.
- Subtotal.

Orders are found by customer relationship OR matching customer email.

This fallback is intentional because some older/backfilled orders may not be perfectly linked to a customer relationship yet.

## Portal Addresses

The first pass of live address data was wired.

The Addresses page now displays billing and shipping records for the signed-in customer.

It can show addresses from:

This fallback prevents the customer portal from appearing empty if older checkouts stored the address on the Order but did not create or link a Customer Address record.

## Download Delivery Foundation

A protected customer download endpoint foundation was added:

```txt
/api/portal/downloads
```

Current behavior:

- Requires the customer to be signed in.
- Checks that the download record belongs to the logged-in customer.
- Blocks inactive or expired download records.
- Does not expose raw R2 object keys.
- Returns a safe “delivery not configured yet” response until signed private file delivery is wired.

Important limitation:

- Actual PDF/EPUB/audiobook file delivery is not complete yet.
- R2/private signed link delivery still needs to be connected.
- The larger automated matching of active Downloads records into Library buttons was attempted but blocked by the connector safety filter, so the safe UI foundation was committed first.
