### Gifting — exact model locked (Hamilton, same session)

Refined `[C] Promotions, Gifting & Access Grants Plan.md` with precise rules:

- A digital license's `maxDownloads` (3) = "up to 3 devices." Gifting spends from the SAME pool: `slotsUsed = downloadsUsed + giftsIssued`; `giftableRemaining = maxDownloads − slotsUsed`. Generating a gift permanently consumes a slot (revoke-if-unredeemed restores it).
- Recipient gets ONE free download, redeemable for any digital product of equal-or-lesser value (value ceiling = gifted item's digital price); not locked to the source book.
- Code: `BPG` + 8 random digits (e.g., BPG48217390); single-use; one download per code per user.
- Portal Gift section generates the code, emails it to the recipient (Mailjet; until then show code for manual share); recipient must sign up to redeem → catalogued as consented lead (`acquired_via: gift`).
- New `gifts` collection (gifter, sourceDownload, sourceBook, format, valueCeiling, redemptionCode, status, downloadsGranted=1, redeemedBy/redeemedBook/redeemedAt, expiresAt). Add `giftsIssued` to downloads; download endpoint enforces personal downloads = `maxDownloads − giftsIssued`.
- Still to confirm: expiry window (suggest 90d), no re-gifting, ship portal Gift now with on-screen code + add email when Mailjet clears.

### Gifting — final code + admin decisions (Hamilton)

- Gift code = total **5–8 chars** = `BPG` + 2–5 random digits (randomized length), e.g. `BPG12345`. `BPG` prefix distinguishes gift codes from admin discount codes (discount codes never use BPG). Unique w/ collision retry; single-use; one download per code per user.
- Gift expiry: **90 days**. No re-gifting (redeemed grants non-transferable).
- Ship order: portal Gift section first with on-screen code + copy/share; automated recipient email added when Mailjet clears.
- Admin: create gift codes directly (auto-random or custom, normalized to `BPG…`); full tracking view of all gift codes (status, code, gifter, recipient, value ceiling, redeemed-by, dates) + revoke unredeemed.
