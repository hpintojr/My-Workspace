### Payload Admin Blank Panel Debugging

Confirmed:

- Raw Neon query returns 9 books through `/api/debug-books`.
- Payload API returns 9 books through `/api/debug-payload-books`.
- Therefore, the backend data layer works.
- The remaining admin issue is the Payload Admin UI/rendering layer.

Current suspected cause:

- Payload Admin was being wrapped by the public website root layout.
- Payload's `RootLayout` renders its own `<html>` and `<body>`.
- The old shared `app/layout.tsx` also rendered `<html>` and `<body>`.
- This likely caused invalid nested root markup and React hydration error `#418`.

Fix direction committed:

```txt
app/(frontend)/layout.tsx  → public website layout and CartProvider
app/(payload)/layout.tsx   → Payload Admin layout only
```

Also moved public pages into the `(frontend)` route group and removed the old shared root layout.

Important QA warning:

- Some public pages were moved quickly during debugging.
- Before production deploy/launch, verify all public routes still have full content, especially Privacy Policy, Terms, For Parents, and Thank You pages.
- Restore full legal page text from Git history if any simplified placeholder content remains in `(frontend)`.
