### Payload Admin Blank Panel Debugging

Confirmed:

- Raw Neon query returns 9 books through `/api/debug-books`.
- Payload API returns 9 books through `/api/debug-payload-books`.
- Therefore, the backend data layer works.
- The admin issue was the Payload Admin UI/rendering layer and then database preference schema drift.

Root causes discovered during the session:

- Payload Admin had been wrapped by the public website root layout.
- Payload's `RootLayout` renders its own `<html>` and `<body>`.
- The old shared `app/layout.tsx` also rendered `<html>` and `<body>`.
- This likely caused invalid nested root markup and React hydration error `#418`.
- Payload Admin also needed repaired preference table schema because it attempted to insert into `payload_preferences_rels.order`.
- Admin CSP blocked `eval` and then blocked Google Fonts until CSP headers were updated.

Fixes committed:

```txt
app/(frontend)/layout.tsx  → public website layout and CartProvider
app/(payload)/layout.tsx   → Payload Admin layout only
next.config.mjs            → admin CSP updates
setup-payload-preferences  → repaired preference relation order column
```

Current result:

```txt
/admin works
/admin/login works
/admin/collections/books works
9 books display in Payload Admin
individual Book edit pages open
```
