### Stripe Fulfillment Recovery — Late Night Fixes

A checkout payment succeeded in Stripe but did not initially appear in Payload Orders. The thank-you page displayed a fallback confirmation reference instead of a real order number because fulfillment failed.

Root causes found:

1. The thank-you page only shows a real order number if fulfillment returns a summary.
2. Manual reconcile was originally run against the wrong URL (`/thank-you?...&secret=...`) instead of the protected API route.
3. Stripe customer email handling triggered a Payload `users.email` validation failure.
4. Customer/User creation failure was blocking the entire order from being created.
5. Shipping data was not where the code expected it. Stripe returned shipping under:

```txt
collected_information.shipping_details
```

instead of only:

```txt
shipping_details
```
