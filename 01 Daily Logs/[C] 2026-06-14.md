## Late-Session Admin Panel QA / Polish

After the dashboard and collections were functional, a second admin-polish pass focused on page-by-page visual QA and Payload native UI conflicts.

### Confirmed / Fixed

- Orders detail pages now open after diagnosing the server-side issue.
- The Orders blank-page issue was not CSS. Vercel logs showed Payload querying `payload_locked_documents_rels.privacy_requests_id`, which did not exist.
- Added website repo documentation for the required Payload locked-document repair:

```txt
docs/PAYLOAD_LOCKED_DOCUMENTS_SCHEMA_PATCH.md
```

- Media link previously returned 404 because the `downloads` collection was hidden. It was exposed and relabeled as Media.
- Users and Customers were separated by sidebar URL behavior:
  - Customers opens Users filtered to `role = customer`.
  - Users opens the full Users collection.
- Users collection labels were changed from Settings to Users.
- Sidebar active state was fixed so Dashboard is not always highlighted.
- Native Payload sidebar/current page labels were hidden so only the custom Adventure Hub nav is visible.
- Sidebar branding was re-centered after active-state changes shifted the Benny & Penny Admin Panel logo/title.
- Subscribers `Marketing Opt In` now renders as `Yes`/`No` instead of raw `true`/`false`.
- Yes/No display was softened to match normal table typography.
- Payload row-selection checkboxes were accidentally affected by broad admin CSS selectors. The source was narrowed to broad `[class*='select']`, collection button styling, and `opacity: revert !important` on checkbox resets.
- Removed broad `[class*='select']` styling because Payload row checkboxes use `select-row` classes.
- Removed `opacity: revert !important` from row checkbox controls after Hamilton identified it as the exact issue.
- Added global dark Payload notification/toast styling because the logout notification appears to render through a portal rather than inside `.template-login`.
